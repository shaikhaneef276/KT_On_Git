Key Concept

Pushing into develop – What Happens?

✅

Clean Push (No Conflict)

# On feature/login branch
git push origin feature/login
# Open PR → develop
# Reviewer approves → Merged!

Your code is merged into develop smoothly. CI/CD pipeline may auto-trigger tests.

⚠️

Merge Conflict

# Dev B also edited style.css
# Git can't auto-merge
CONFLICT (content): Merge conflict
in style.css

You must resolve manually, commit the fix, then the PR can be merged.

🔒

Protected Branch Rule

# NEVER do this!
git push origin develop
# Use PRs always!
git push origin feature/login

Direct push to develop is blocked. Always go through Pull Requests for code review.

Conflict Resolution

Resolving Merge Conflicts

1
Git marks conflict 
<<<<<<< HEAD (develop)
body { background: #fff; }
=======
body { background: #000; }
>>>>>>> feature/login
↓
2
You decide what stays 
body { background: #1a1a2e; }
/* Agreed-upon value */
↓
3
Commit the resolution 
git add style.css
git commit -m "resolve: merge conflict in style.css"
git push origin feature/login
💡 Prevention: Pull from develop frequently — fewer conflicts!

Rules

Dos & Don'ts

✅ DO
  • Always branch off develop
  • Pull latest develop before starting
  • Use descriptive branch names
  • Write meaningful commit messages
  • Open a PR for code review
  • Delete branch after merge
  • Communicate conflicts with team
❌ DON'T
  • Push directly to main or develop
  • Commit with message "fix" or "wip"
  • Force push to shared branches
  • Merge without review
  • Work on main for days without pushing
  • Ignore merge conflicts
  • Skip pulling before starting work
Recap

Quick Cheat Sheet

🚀 Start new feature
git checkout develop
git pull origin develop
git checkout -b feature/my-task
💾 Save your work
git add .
git commit -m "feat: describe change"
git push origin feature/my-task
🔄 Sync with team
git pull origin develop
# Fix conflicts if any
git push origin feature/my-task
🧹 After PR merged
git checkout develop
git pull origin develop
git branch -d feature/my-task
🎉 You're ready to collaborate! Remember: Branch → Code → Commit → Push → PR → Merge
